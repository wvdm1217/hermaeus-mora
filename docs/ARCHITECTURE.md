# Hermaeus Mora Architecture

The agentic prince of knowledge and memory.

## Architectural Vision

`hermaeus-mora` is a Python package designed to bring order to personal knowledge and memories. At its core, all data is stored as plain, human-readable, and easily editable Markdown files.

Raw notes and scribblings are often chaotic and lack immediate utility, but they contain hidden insights. The goal of Hermaeus Mora is to provide a system where these fragmented thoughts can be continuously refined over time. To be truly useful, memories must maintain temporal coherence, form semantic links with one another, and be highly discoverable through both vector and full-text search. Ultimately, the system ensures that the right information surfaces precisely when it is needed for a specific task or question.

## Core Mechanisms

While there are many existing memory systems, Hermaeus Mora prioritizes the interface—what the user and the agent actually interact with.

- **Markdown-Native Interface**: Because memories are stored as Markdown, any standard coding agent or text editor can read, modify, and refine them without specialized APIs.
- **Continuous Synchronization**: Every time a memory is created or edited, the changes are automatically ingested into an underlying vector database and a knowledge graph system.
- **Agentic Refinement**: Specialized agents can autonomously review, correct, and interlink incomplete or incorrect memories. The system seamlessly supports both agent-generated and human-generated memories.
- **Interactive Memory Construction**: Users can engage with Hermaeus Mora to construct new memories in a dynamic, canvas-like interface. By answering questions and interacting with the agent, users can naturally flesh out scribbles into structured knowledge with minimal formatting friction.

## Data Sources & Curation

To provide maximum contextual awareness, users can connect incoming data streams such as messages, emails, and calendar meetings. Rather than blindly storing everything, the system acts as a curation engine, identifying and keeping track of the things that are genuinely worth remembering.

A major focus of this curation is **human interactions**. By tracking meetings, communications, and shared contexts, Hermaeus Mora builds up comprehensive profiles of people over time. This empowers the user with the exact historical context needed to organize their life and respond thoughtfully to others.

## Memory Hierarchy

To maintain signal over noise, memories are structured hierarchically:

1. **User Memories**: The foundational, most important layer. These are explicit thoughts, notes, and records created by the user.
2. **Agent Memories**: Insights, facts, and synthesis generated by agents based on user input and environmental context.
3. **Interactions**: Stored logs of user-agent conversations and actions, providing contextual history.

## Metadata System

A robust metadata schema is essential for temporal tracking, searchability, and graph linking. This metadata is embedded directly into the Markdown files as YAML frontmatter:

```markdown
---
id: 1234
created_at: 2024-06-01T12:00:00Z
updated_at: 2024-06-01T12:00:00Z
tags: [tag1, tag2]
links: [5678, 91011]
summary: "A brief summary of the memory content."
source: "Original source of the memory (e.g., book, conversation, personal experience)."
importance: 5  # Scale of 1-10 indicating the importance of the memory.
context: "Additional context or background information relevant to the memory."
---
# Memory Content
This is the main content of the memory, which can be a detailed description, notes, or insights related to the memory. It can include text, images, or other media as needed.
```
